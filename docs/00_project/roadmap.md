# プロダクトロードマップ(全4スプリント)

- **プロダクト**: CRUD 機能を持つ TODO アプリ
- **技術スタック**: Docker(bitnami/laravel)+ Laravel + MySQL + Plain HTML/CSS
- **1 スプリント**: 想定作業時間 3〜4 時間
- **完成の定義(プロジェクト全体)**: TODO の作成・一覧・詳細・編集・完了切替・削除がブラウザから操作でき、ドキュメントと共に GitHub に公開されている

## 全体の見通し

| スプリント | テーマ | 主なゴール |
|---|---|---|
| Sprint 1 | 環境構築と土台づくり | Docker 上で Laravel + MySQL が起動し、ブラウザで初期ページが見える。GitHub リポジトリ整備 |
| Sprint 2 | Create + Read(一覧) | TODO を新規作成でき、一覧に表示される。バリデーション実装 |
| Sprint 3 | Read(詳細)+ Update | TODO の詳細表示・編集・完了状態の切り替えができる |
| Sprint 4 | Delete + 仕上げ | TODO を削除できる。CSS を整え、全機能の受け入れ確認をして完了 |

## 各スプリントの狙い(学習テーマ)

### Sprint 1: 環境構築と土台づくり
- Docker / Docker Compose の基本(コンテナ、イメージ、ポート)
- Laravel プロジェクトの構造(MVC の全体像)
- Git / GitHub Flow の運用開始

### Sprint 2: Create + Read(一覧)
- マイグレーション(DB のテーブル定義をコードで管理する仕組み)
- Eloquent モデル(DB のテーブルを PHP のクラスとして扱う仕組み)
- ルーティング → コントローラ → ビューの流れ
- フォーム送信・バリデーション・CSRF 対策

### Sprint 3: Read(詳細)+ Update
- ルートパラメータとモデルバインディング(URL の ID から自動でレコードを取る仕組み)
- 編集フォームと更新処理(PUT/PATCH メソッド)
- 完了状態のトグル処理

### Sprint 4: Delete + 仕上げ
- 削除処理(DELETE メソッド)と誤操作の防止
- CSS での見た目調整(フレームワークなし)
- 受け入れテスト(全ユーザーストーリーの確認)と振り返り

## スコープ外(今回はやらないこと)

- 認証(ログイン)機能
- 期限日・優先度などの追加項目
- JS / CSS フレームワークの導入
- 自動テスト(手動での動作確認を DoD とする)
