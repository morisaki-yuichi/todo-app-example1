# ユーザーストーリー

形式: 「〜として、〜したい、なぜなら〜だからだ」

今回のユーザーは「自分のタスクを管理したい個人ユーザー」1 種類のみ。

## ストーリー一覧

### US-1: TODO を作成する(Create)
> タスクを管理したいユーザーとして、タイトルと内容を入力して TODO を登録したい。なぜなら、やるべきことを忘れないように記録しておきたいからだ。

**受け入れ条件**
- タイトル(必須・100 文字以内)と内容(任意・1000 文字以内)を入力して保存できる
- タイトルが空、または文字数超過のときは日本語のエラーが表示され、入力値が残る
- 保存後は一覧画面に戻り、登録した TODO が表示されている

### US-2: TODO の一覧を見る(Read)
> タスクを管理したいユーザーとして、登録した TODO を一覧で見たい。なぜなら、今やるべきことを一目で把握したいからだ。

**受け入れ条件**
- 登録済みの TODO がタイトル・完了状態つきで一覧表示される
- 作成日時の新しい順に並ぶ
- 1 件もないときは「TODO がありません」の旨が表示される

### US-3: TODO の詳細を見る(Read)
> タスクを管理したいユーザーとして、TODO の詳細(内容メモ)を見たい。なぜなら、一覧では省略される詳しい情報を確認したいからだ。

**受け入れ条件**
- 一覧から 1 件を選ぶと、タイトル・内容・完了状態・作成/更新日時が表示される
- 存在しない ID にアクセスすると 404 になる

### US-4: TODO を編集する(Update)
> タスクを管理したいユーザーとして、登録済みの TODO のタイトルや内容を書き換えたい。なぜなら、予定や内容は後から変わることがあるからだ。

**受け入れ条件**
- 編集フォームに現在の値が入った状態で表示される
- バリデーションは作成時と同じルールが適用される
- 保存後、変更が一覧・詳細に反映されている

### US-5: TODO を完了/未完了にする(Update)
> タスクを管理したいユーザーとして、終わった TODO に完了マークを付けたい(外したい)。なぜなら、残っているタスクと終わったタスクを区別したいからだ。

**受け入れ条件**
- 一覧または詳細からワンクリックで完了⇔未完了を切り替えられる
- 完了した TODO は見た目で区別できる(打ち消し線など)

### US-6: TODO を削除する(Delete)
> タスクを管理したいユーザーとして、不要になった TODO を削除したい。なぜなら、やらないことが一覧に残っていると邪魔だからだ。

**受け入れ条件**
- 削除前に確認が入る(誤操作防止)
- 削除後は一覧から消えている
