## 優先順位の考え方

1. **動く土台が最優先**(#1〜#2): 環境がなければ何も作れない
2. **Read より先に Create は作れない、Create より先に一覧が欲しい**(#3〜#5): データを入れる手段と見る手段はセットで 1 スプリントに置く
3. **Update 系**(#6〜#8): 既存データがあって初めて意味を持つ
4. **Delete と仕上げ**(#9〜#11): 最後に品質を上げて完成させる

## 完了の定義(DoD: Definition of Done)

各 PBI は以下をすべて満たしたら「完了」とする。

- [ ] ブラウザで実際に動作確認できた
- [ ] 受け入れ条件をすべて満たしている
- [ ] コードが `main` ブランチにマージされている
- [ ] 必要なドキュメントが更新されている
