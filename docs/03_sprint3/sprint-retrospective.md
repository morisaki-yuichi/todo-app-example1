## 学びの要約(開発者自身の言葉で)

### 学び1: ルートモデルバインディングは「定型コードの自動化」
`/todos/{todo}` と型付き引数 `Todo $todo` を組み合わせると、「ID で検索 → なければ 404」を Laravel が肩代わりしてくれる。存在しない ID の 404 も自分で書かなくてよい。フレームワークの機能は「手で書くと面倒な定型処理の自動化」と捉えると学びやすい。

### 学び2: HTMLフォームの制約とHTTPメソッドの使い分け
REST では「更新は PUT/PATCH、削除は DELETE」だが、HTML フォームは GET/POST しか送れない。`@method('PUT')` が隠しフィールドで補ってくれる。また「GET でデータを変えてはいけない」— 完了切替をリンクにすると、ブラウザの先読みだけで TODO が完了になる事故が起きうる。

### 学び3: 「エラーが出ない不具合」の切り分けは外側から順に
タイムゾーンが直らなかったとき、「環境変数はコンテナに届いているか → config は環境変数を読んでいるか」と、データが流れる順に確認して原因(config の直書き)に到達した。エラーメッセージがない不具合ほど、経路を1段ずつ確かめるのが近道。

### 学び4: 同じルールは1箇所に(DRY)
作成と編集のバリデーションをコピペせず `validateTodo()` に共通化した。ルールが2箇所にあると、将来の変更時に片方だけ直して不整合になる。「同じものを2回書いたら共通化を考える」を習慣にする。
