## スプリント中の気づきと対処

### タイムゾーンのズレ(計画外の品質改善)
- 詳細画面の日時が UTC 表示(9 時間ズレ)であることをデモ確認中に発見
- `APP_TIMEZONE=Asia/Tokyo` を compose に追加 → **直らない** → 調査
- 「環境変数はコンテナに届いているか(届いていた)→ config が読んでいるか(読んでいなかった)」の順で切り分け
- 原因: `config/app.php` の `'timezone' => 'UTC'` 直書き。`env('APP_TIMEZONE', 'UTC')` に修正
- 教訓: **Laravel の設定は「環境変数 → config → アプリ」の 2 段構え**。config が `env()` を参照していなければ環境変数は無視される

## 実装の要点(学習メモ)

- **ルートモデルバインディング**: 型付き引数 `show(Todo $todo)` だけで取得+404 が自動化
- **ルート定義の順序**: 具体的なパス(/todos/create)が先、可変パス(/todos/{todo})が後
- **メソッドスプーフィング**: HTML フォームは GET/POST のみ → `@method('PUT')` で読み替え
- **GET で状態を変えない**: 完了切替はリンクではなく PATCH フォーム(先読み・クローラ事故の防止)
- **DRY**: バリデーションを `validateTodo()` に集約し、ルール変更が 1 箇所で済むように
